Python needs anonymous functions

2015-05-11
by Paul Agapow
in programming

tagged programming, python, functional programming

Python, you've been good to me and to lots of other people. You've provided a solid, stable base for programming for years. And when change is needed, you didn't rush into it, time was spent thinking about it and a good solution was found.

So why the hell don't you have proper anonymous functions?

An anonymous function is just a function without an identifier, a naked set of parameter signature and function body. There are legion uses for such a thing in functional programming, where it's synatctically lighter and clearer to define and pass a function on the spot, rather than having to explicitly name and define it externally. Ruby has them, C++ has them. Even PHP has them. So why don't you have them Python?

Sure, you could argue that other langauges use them for things that Python doesn't need. For example, Ruby uses its "blocks" for various forms of iteration and context:

ex.sort_by { |x| x   x.to_i }

Instead, Python has for for iteration and with for context. But if I just want to make a functor for readability, or concoct one on the fly, my two options are the extremes of lambdas and full blown function. One is very limited - a single expression, with no assignment. (And reabability can suffer badly when I try and force something into a lambda form.) The other is too heavy and destructive of readability.

Yes, we've got list comprehensions. Which would be so more powerful if we could pass in anonymous functions. yes, I could try and trick a decorator into doing it but in that case I'm just disguising an external function as a decorator.

This is shaping up to be one of those times where Python has an obvious lack, that the fanboys deny exists. ("A quick-and-dity web development system like Ruby-on-Rails? Nah, why would you want that? Anyway, Django is pretty good. Functional programming? Who uses that?")

This is Python's Achilles heel. What we need is a middle ground, a quick way of constructing anonymous functions. And there's no earthly reason why Python couldn't accommodate it.
