Cowboy analysts

2017-02-22
by Paul Agapow
in science

tagged data-science, bioinformatics, humour

Overheard, with my face in my hand

I've previously made fun of how non-analysts expect all sorts of magic to be performed by analysts, instantly and with no resources. It was cathartic, but you have to admit that the semi-mystical "black box" that surrounds informaticists is not helped by their own behaviour: inpenetrable jargon, indifference to other disciplines, and sometimes sheer technical incompetence. While I started writing this with bioinformaticians in mind, it extends to data scientists, statisticians and many other technical diciplines.

"Backups? No, I don't do them. The computer's only a year old, so it should be alright."

"Why would I need to know anything about the problem domain?"

"I wrote my own parser for [common data format for which many good parsers exists already]."

"Version control? Git? I'm the only one working on the code - why would I need that?"

"Version control? Git? We can work together on the code by just emailing it to each other! Or keeping it in Dropbox!"

"Sure, we don't know anything about the range of possible parameter values. But we've got plenty of data and it's a Bayesian analysis. It will work it out."

"What - you did your analysis in Python? Yeah, that's too new and weird. Can't possibly be any good."

"Look at all the time I saved with this solution!"

[Describes spending several weeks to hack together an unmaintainable teetering edifice of software. By the time it crashes or the one of the many security holes are exploited, the creator wil have moved on. But at least you didn't have read the manual for and install the freely available alternative.]

"We grouped similar records ..."

"Using a similarity score ..."

[How? How did you find similar things? How did you measure similarity? How?]

"Students t-test"

"Sure, it's technically impressive and powerful. But if it doesn't work on Windows, I can't / won't use it."

[This actually happened to me. Seriously.]

UH. UM. ER. UH. UM.

"I've just got so many resumes / newsfeeds / job offers to get through, I wrote a recommenddation engine to filter them for me."
