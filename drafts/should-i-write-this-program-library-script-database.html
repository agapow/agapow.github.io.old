Should I write this program / library / script / database?

2015-09-25
by Paul Agapow
in science

tagged bioinformatics, programming, software, science

Do you really want to do this? [y/n]

Before you set out and author a program, build a library, construct a database ... make some sort of re-usable and semi-permanent informatics resource, think about it:

Is it needed?

  • Will this be used in the future? More than twice? Why?
  • Would you have used this in the past if it was available? More than twice? Why?

Is it (actually) needed by other people?

  • Is this one of those occassions where someone says "we could use a database / program that (BLAH BLAH)" and when you finish it, they'll say "Great! Now could you run it and find out ..."
  • Put another way, is anyone but you ever going to use it? Would that web database be better off as an SQlite db or even Excel spreadsheet on your harddrive?
  • Most databases are almost never used. There's something to be said for data preservation, but that could be acheived without a database.
  • You might program in sophisticated search with boolean operators and filters but basically your users will just want to download the whole thing.
  • There's often a strange magic and wishful thinking in the statement "let's make a database ..."
  • ... especially when the data hasn't been collected as yet
  • ... especially when there's a poor idea of the fields or data to be collected, as if this is a mere implementation detail that can be retroactively applied.
  • As noted elsewhere, software is seen by academics as a merely technical / functional / "working class" area. SO what are you getting out of it
